# Security Policy

## Reporting a Vulnerability

We take the security of CrowdWisdomTrading seriously. If you discover a
security vulnerability, please **do not** open a public issue. Instead,
send a private report to the repository maintainers.

Contact: [security@example.com](mailto:security@example.com)

Please include:

- A description of the vulnerability.
- Steps to reproduce it.
- Potential impact.
- Any suggested fix (if known).

You should receive a response within 48 hours. We will keep you informed
of our progress toward a fix.

## Scope

The following are considered in-scope for security reports:

- Exposure of API keys, database credentials, or other secrets.
- SQL injection or other database attacks.
- Arbitrary code execution via crafted input files.
- Path traversal in file I/O operations.

## Out of Scope

- Algorithmic weaknesses in the trading model (these are research issues).
- Performance problems (open a regular issue).

## Preferred Languages

We prefer reports in English. Reports in other languages may take longer
to process.
